# The elevator ride (draft)

This note connects **personal pacing** to the **structured system** in this workspace. It is not a product spec; it is an **ambiance anchor** for how to move without confusing the map with the ride.

## Floors (system)

- **Lobby** — [`CENTRAL_PLAZA.md`](../CENTRAL_PLAZA.md): where you *say* you are (districts, health language).
- **Shaft** — [`SPEC.md`](../SPEC.md) planes + bus: *roles* (services, runs, artifacts, surfaces).
- **Cab controls** — gates ([`WORKSPACE_GATES.md`](../WORKSPACE_GATES.md)), routines under `racks/routines/`, MCP tools under `CascadeProjects/`.
- **Machine room** — submodule `CascadeProjects/` (hogsmade), schedules, CI, ships.

You do not live on one floor only. You **ride** between them: docs-only passes, integration passes, research passes.

## Curvature in the lap

A **lap** is one full cycle through your chosen loop: orient, touch the repo, produce an artifact, reconcile. **Curvature** is where the track bends—the places people lose time: ambiguous paths, stale symlinks, mixed vocabulary (plaza vs plane), or treating agent output as deployable truth without a gate.

Steering is **repeating the same structured loop** until momentum carries you: small commits, clear run IDs, audit lines when tools fire, proportion JSON when rehearsing GRUFF bridges—not because the loop is glamorous, but because **schedule + repetition** beats heroic one-offs.

## Personal experience inside structure

Agentic tools and cognitive assistants produce **streams**. This repo expects you to **dock** those streams: review packages, manifests, hooks—not to replace human judgment, but to stop drift. When something “feels” insightful, **name the floor** it belongs on (plaza story vs plane contract vs artifact file) before you encode it.
