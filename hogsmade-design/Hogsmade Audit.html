1 · summary

short version, before the details

what's working

  • Clear hub exists — hogsmade/CascadeProjects acts as the umbrella workspace. 12 MCP servers are correctly namespaced under Tools/MCPServers/.
  • GRID is correctly a submodule, not a copy — single source of truth.
  • Shared packages (shared-types, shared-resilience, shared-pipeline) sit in Components/ with a stated build order. Good.
  • Contracts + envelopes + audits have a shared schema (GATE, echoes, contract.hardened.json pattern across afloat & aac).

what's not

  • Three homes for "afloat": repo afloat, local canopy/afloat, and Tools/MCPServers/afloat-server — a Next.js app, a duplicate, and an MCP server, all sharing a name. Routing is ambiguous.
  • Duplicate manifest — canopy/assistive-agreement-contracts/manifest.json still claims "name":"afloat". Copy-paste bleed.
  • Bus is described, not implemented. 12 MCP servers but no central event stream — the "central hub" is just a folder, not a runtime.
  • glimpse-* is split into artifact/engine AND has an MCP server AND has code inside echoes. 3 owners of "Glimpse" terminology.
  • Orphans: apiguard, python-craft, Atmosphere, storage, light_of_the_seven, magical_forest, concept have no declared seat in the 4-planes model.
  • Vision vs GRID overlap — both host "learning_data", both LLM-aware, no contract boundary between them.
