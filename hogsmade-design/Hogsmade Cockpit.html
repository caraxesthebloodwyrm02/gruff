A · palette + planes

cmd-K centric · 3-col · dense but readable
sidebar = "what exists". main = "what I'm doing". rail = "what just happened". command bar is the primary input for everything — servers, runs, artifacts search the same index.

B · terminal-first

one big REPL · panes open on demand
pure keyboard. great for power users. hard for newcomers — every concept is hidden behind a verb. rejected as default; kept as a mode (press `).

C · graph / dependency map

see the whole system as a topology
beautiful for explaining the system once. bad for daily work — nodes don't do anything except open detail. kept as an optional "map" tab.

D · classic tabs

one tab per plane · familiar · flat
safe and boring. we get it for free by toggling layout — but loses context: you can't see runs while reading a service.

E · governance board (Hogwarts flavor)

houses = planes · arena = runs · board = decisions
on-brand and fun. but the metaphor taxes newcomers — "what's a seconder?" adds cognitive cost for a cockpit. great as a skin toggle over layout A.
